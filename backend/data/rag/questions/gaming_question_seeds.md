# Mobile Gaming Interview Question Seeds

## Gameplay and client engineering

- Describe a performance problem on mobile devices and how you measured the fix.
- How do you structure game state for levels, progression, and live config updates?
- Explain asset loading strategy to reduce cold start and level load time.
- How would you implement a daily reward system without clock-skew exploits?
- What causes frame drops in Unity/mobile engines and how do you profile them?

## Live ops and content

- How do you ship a live event safely without increasing crash rate?
- Describe remote config / feature flag strategy for game balance changes.
- Tell me about an A/B test on tutorial, economy, or difficulty—guardrails and outcome.
- How do you roll back a bad content bundle or config push?
- What metrics define live ops success (D1/D7, session length, crash-free)?

## Backend and analytics (Peak, Dream Games-style)

- Design leaderboard system with cheat prevention for async puzzle game.
- How do you ingest client analytics events reliably with offline players?
- Explain matchmaking or league placement at high level for casual competitive mode.
- How would you detect economy inflation or reward exploit in production?

## Behavioral

- Tell me about a launch that hurt retention—what did you learn and change?
- Describe conflict between game design and engineering constraints on deadline.
- Share a crash spike after release—timeline from detection to fix.
- How do you prioritize polish vs new features before global launch?

## Adaptive follow-ups

- What was crash-free rate before and after?
- How did you validate on low-end devices common in target market?
- What guardrail metric stopped a bad experiment from full rollout?
- Personal contribution vs team—what did you specifically build or decide?
