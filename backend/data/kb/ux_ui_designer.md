# UX/UI Designer — Interview Coaching Knowledge Base

## Role overview

UX/UI designer interviews evaluate research rigor, interaction design quality, visual craft, systems thinking, and collaboration with product and engineering. Portfolio review often dominates; behavioral and whiteboard prompts test process, tradeoffs, and accessibility—not only aesthetic taste.

## Core competencies

- Research: interviews, surveys, usability tests, synthesis into insights
- IA and flows: task flows, wireframes, edge cases, error states
- Interaction design: affordances, feedback, consistency, mobile patterns
- Visual design: typography, hierarchy, spacing, design tokens, brand alignment
- Design systems: components, documentation, contribution model
- Accessibility: WCAG-oriented color contrast, focus order, screen reader considerations

## Interview themes

Walk through design process on recent project, validate decisions with users, design system contribution, handle contradictory stakeholder feedback, accessibility approach. App critiques and " redesign this screen" exercises test structured critique, not hot takes.

## Question bank

**Portfolio deep dive:** Problem, users, constraints, iterations, validation, outcome metrics.

**Whiteboard:** Onboarding for fintech app, checkout error states, settings for elderly users.

**Behavioral:** Stakeholder wanted bad UX for business reasons—response. Engineering said design not feasible—collaboration. Usability test surprised you.

**Systems:** Component you added to design system and why. Document for handoff to developers.

## Strong answer framework

Problem → users/research → constraints → explorations (show why rejected) → final design → validation (qual + quant if any) → shipped outcome or learning. Critique others' work with heuristic structure: clarity, hierarchy, accessibility, consistency.

## Strong answer signals

- Research insight drove design change, not post-hoc justification
- Edge cases designed: empty, error, loading, partial permissions
- Accessibility considered early, not audit at end
- Developer handoff: specs, redlines, tokens, interaction notes
- Metrics: task completion, SUS, support tickets, conversion if available

Example strong fragment: "Usability tests showed users missed optional KYC step—we mistaken affordance for progress. Added explicit stepper and primary CTA copy change. Task completion rose from 61% to 84% in moderated retest before dev sprint."

## Weak answer signals and red flags

- Pixel pushing without user problem
- No iteration story; first design was "final"
- Cannot explain accessibility beyond alt text cliché
- Dismisses engineering constraints arrogantly
- Portfolio only Dribbble shots without process or outcomes

## Common interviewer follow-ups

- What would you test next and how?
- How do you prioritize UX debt?
- Mobile vs desktop tradeoffs for this flow?
- How do you measure design success post-launch?
- How do you work with PM on scope cuts?

## Seniority signals

**Junior:** Executes flows from briefs. **Mid:** Owns feature UX end-to-end with research. **Senior:** Shapes design system, research practice, cross-product IA, mentors craft.

## Coaching tips

Prepare one project with clear before/after metric or research quote. Critique exercises: start with user goal, not "I don't like the color." For bank or airline clients, emphasize trust, clarity under stress, and accessible defaults.
## Example answer snippets

**Weak:** "I made a clean modern UI in Figma."
**Strong:** "Loan application abandonment spiked on income field. Moderated tests showed users unsure whether net or gross income was requested. Revised label, helper text, and inline validation example; completion on that step improved 28% in A/B."

## Local market context

Bank and fintech UX in Turkey requires trust, clarity on fees, and accessible Turkish copy. Aviation and travel apps need stress-case flows (delays, rebooking). Show research evidence, not only visual polish.
## Deep-dive topic bank

Review: heuristic eval; cognitive load; form design; error microcopy; responsive strategy; purposeful motion; handoff checklist; inclusive design; diary vs usability tests; SUS and task time metrics.
